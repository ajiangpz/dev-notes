`<meta name="viewport">` 是移动端网页开发中**非常重要的一个标签**，它的主要作用是**控制网页在移动设备上的视口（viewport）大小和缩放比例**，从而影响页面的布局和显示效果。

---

## 🧠 一句话概括

> `<meta name="viewport">` 告诉浏览器**页面在手机上该怎么显示**：是按 PC 那样缩放，还是适配手机屏幕宽度、禁止缩放、初始放大多少。

---

## 🔍 详细解释

### 📱 什么是 viewport？

- \*\*Viewport（视口）\*\*是浏览器中网页的可见区域。
- 在**桌面浏览器**中，viewport 通常就是浏览器窗口的宽度。
- 但在**移动设备**上，默认 viewport 会比屏幕宽很多（通常是 980px 或更宽），为了显示完整内容，浏览器会自动缩小页面，看起来“字很小”。

---

## 🧾 标签语法示例

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
```

### 👇 这是最常用的写法，表示：

| 属性                 | 作用                                     |
| -------------------- | ---------------------------------------- |
| `width=device-width` | 视口宽度等于设备屏幕宽度（避免默认缩放） |
| `initial-scale=1.0`  | 初始缩放比例为 1（即不放大不缩小）       |

---

## 📋 `content` 可配置项详解

| 参数            | 作用                   | 示例值                                         |
| --------------- | ---------------------- | ---------------------------------------------- |
| `width`         | 设置视口宽度           | `device-width`（设备宽度）或固定值（如 `640`） |
| `height`        | 设置视口高度（很少用） | `device-height`                                |
| `initial-scale` | 初始缩放比例           | `1.0`                                          |
| `minimum-scale` | 最小缩放比例           | `0.5`                                          |
| `maximum-scale` | 最大缩放比例           | `2.0`                                          |
| `user-scalable` | 是否允许用户缩放       | `yes` / `no`                                   |

### 💡 举个例子：

```html
<meta
  name="viewport"
  content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no"
/>
```

表示：

- 视口宽度等于设备宽度；
- 初始缩放为 1；
- 不允许放大缩小；
- 最大缩放也限制为 1（配合 `user-scalable=no`，常用于不希望用户缩放的页面）。

---

## 📊 对比效果（有 vs 无 viewport）

| 场景       | 无 viewport 标签               | 有 viewport 标签                   |
| ---------- | ------------------------------ | ---------------------------------- |
| 手机上显示 | 页面会按 PC 宽度渲染并整体缩小 | 页面宽度适配屏幕宽度，字体正常显示 |
| 交互体验   | 字小，需手动缩放               | 字体清晰，用户体验好               |
| 布局效果   | 布局被压缩或错乱               | 正常响应式布局                     |

---

## 📱 常见应用场景

1. **响应式网页（Responsive Design）**：必须配合 viewport 才能正常生效。
2. **移动端 Web App**：通常会禁止缩放以避免 UI 错乱。
3. **H5 页面 / 微信内嵌页**：几乎都要设置该标签，否则字体非常小。

---

## ✅ 总结

| 要点           | 说明                                                                     |
| -------------- | ------------------------------------------------------------------------ |
| 作用           | 控制移动端网页视口大小和缩放                                             |
| 必须性         | 移动端网页基本都需要                                                     |
| 常用写法       | `<meta name="viewport" content="width=device-width, initial-scale=1.0">` |
| 配合响应式设计 | 是响应式布局的基础前提                                                   |
