# JavaScript 变量和作用域

## 基本问题

### 1. JavaScript中有哪些声明变量的方式？

**答案**：JavaScript中有三种声明变量的方式：

- **var**：函数作用域，存在变量提升
- **let**：块级作用域，不存在变量提升，不允许重复声明
- **const**：块级作用域，声明常量，必须初始化且不能重新赋值

### 2. 什么是变量作用域？

**答案**：变量作用域是指变量在代码中的可访问范围。JavaScript中主要有以下几种作用域：

1. **全局作用域**：在代码的任何地方都能访问的变量
2. **函数作用域**：在函数内部定义的变量，只能在函数内部访问
3. **块级作用域**：ES6引入，使用let和const声明的变量具有块级作用域
4. **词法作用域**：函数的作用域在函数定义时确定，而非调用时

### 3. var、let、const的区别？

**答案**：主要区别包括：

1. **作用域**：
   - var: 函数作用域
   - let/const: 块级作用域

2. **变量提升**：
   - var: 会提升，初始值为undefined
   - let/const: 不会提升，存在暂时性死区

3. **重复声明**：
   - var: 允许重复声明
   - let/const: 不允许重复声明

4. **全局对象属性**：
   - var: 声明的变量会成为全局对象属性
   - let/const: 不会

## 进阶问题

### 1. 什么是暂时性死区（TDZ）？

**答案**：暂时性死区是指使用let/const声明的变量在声明之前不能被访问的现象。

### 2. 如何理解词法作用域？

**答案**：词法作用域是指变量的作用域是在代码编写时就确定的，而不是在运行时确定。它遵循函数定义时的作用域链。

### 3. 闭包与作用域的关系？

**答案**：闭包是基于词法作用域的特性，允许函数访问其定义时的作用域，即使在其他作用域中执行。
