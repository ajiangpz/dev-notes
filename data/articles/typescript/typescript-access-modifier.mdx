在 TypeScript 中，支持以下 **访问修饰符（Access Modifiers）** 来控制类成员（属性、方法）的可见性：

---

### 1. `public`（公开）

* **默认修饰符**，不写时就是 `public`。
* 成员在任何地方都可以被访问。

```ts
class Person {
  public name: string;

  constructor(name: string) {
    this.name = name;
  }

  public greet() {
    console.log(`Hello, I'm ${this.name}`);
  }
}
```

---

### 2. `private`（私有）

* 只能在类的内部访问，**子类和外部都无法访问**。

```ts
class Person {
  private age: number;

  constructor(age: number) {
    this.age = age;
  }

  private getAge() {
    return this.age;
  }
}
```

---

### 3. `protected`（受保护）

* 只能在类内部和**子类中访问**，外部无法访问。

```ts
class Person {
  protected gender: string = 'unknown';
}

class Student extends Person {
  showGender() {
    console.log(this.gender); // ✅ 可访问
  }
}

const p = new Person();
// console.log(p.gender); // ❌ 报错，外部无法访问
```

---

### 4. `readonly`（只读）*（不是访问修饰符，但常一起使用）*

* 只能在声明时或构造函数中赋值，之后不能修改。

```ts
class Person {
  readonly id: number;

  constructor(id: number) {
    this.id = id;
  }

  updateId(newId: number) {
    // this.id = newId; // ❌ 报错，readonly 属性不能修改
  }
}
```

---

### 小结对比表：

| 修饰符         | 类内部可访问 | 子类可访问 | 类外部可访问 |
| ----------- | ------ | ----- | ------ |
| `public`    | ✅      | ✅     | ✅      |
| `protected` | ✅      | ✅     | ❌      |
| `private`   | ✅      | ❌     | ❌      |
